# Getting Started with DevExpress Word Processing Document API (.NET Framework)

This guide covers setup and usage of the Word Processing Document API in .NET Framework 4.6.2+ projects.

## When to Use This Reference

Use this when you need to:
- Target .NET Framework 4.6.2, 4.7, or 4.8 specifically
- Reference DevExpress assemblies directly instead of using NuGet packages
- Understand differences between .NET and .NET Framework usage
- Create a WinForms, WPF, or ASP.NET (classic) project with Word processing capabilities

## System Requirements

- .NET Framework 4.6.2 or later
- Visual Studio 2019+ or JetBrains Rider
- DevExpress Unified Component Installer (for direct assembly references) **or** the DevExpress NuGet feed
- A valid DevExpress license

## Installation Option 1: NuGet Package (Recommended)

Add the DevExpress NuGet feed and install:

**Package Manager Console:**
```
Install-Package DevExpress.Document.Processor
```

This pulls in all required dependencies automatically.

## Installation Option 2: Direct Assembly References

If you have the DevExpress Unified Component Installer, add references to these DLLs (replace `<version>` with your DevExpress version, e.g., `v25.2`):

| Assembly | Purpose |
|----------|---------|
| `DevExpress.Data.v<version>.dll` | Core data layer |
| `DevExpress.Drawing.v<version>.dll` | Graphics and drawing |
| `DevExpress.Office.v<version>.Core.dll` | Office document model |
| `DevExpress.RichEdit.v<version>.Core.dll` | Word processing engine |
| `DevExpress.Printing.v<version>.Core.dll` | Printing and PDF rendering |
| `DevExpress.Pdf.v<version>.Core.dll` | PDF export |

Also add a reference to **System.Drawing** from the framework.

> **Important**: All referenced DevExpress assemblies must be the same version.

## Your First Document (.NET Framework)

```csharp
using DevExpress.XtraRichEdit;
using DevExpress.XtraRichEdit.API.Native;
using System.Drawing;
using System.Diagnostics;

static void Main(string[] args)
{
    string fileName = "Test.docx";
    using (var wordProcessor = new RichEditDocumentServer())
    {
        Document doc = wordProcessor.Document;
        doc.AppendText("This document is generated by Word Processing Document API");

        CharacterProperties cp = doc.BeginUpdateCharacters(doc.Paragraphs[0].Range);
        cp.ForeColor = Color.FromArgb(0x83, 0x92, 0x96);
        cp.Italic = true;
        doc.EndUpdateCharacters(cp);

        ParagraphProperties pp = doc.BeginUpdateParagraphs(doc.Paragraphs[0].Range);
        pp.Alignment = ParagraphAlignment.Right;
        doc.EndUpdateParagraphs(pp);

        wordProcessor.SaveDocument(fileName, DocumentFormat.Docx);
    }
    // Open the file with the default application
    Process.Start(fileName);
}
```

> **Note**: On .NET Framework, `Process.Start(fileName)` opens the file directly. On .NET 6+, use `Process.Start(new ProcessStartInfo(fileName) { UseShellExecute = true })`.

## Load and Save (.NET Framework)

```csharp
using (var server = new RichEditDocumentServer())
{
    server.LoadDocument("input.docx", DocumentFormat.Docx);
    Document doc = server.Document;

    // Modify content
    ParagraphProperties pp = doc.BeginUpdateParagraphs(doc.Paragraphs[0].Range);
    pp.Alignment = ParagraphAlignment.Center;
    doc.EndUpdateParagraphs(pp);

    server.SaveDocument("output.docx", DocumentFormat.Docx);
}
```

## Key Differences: .NET Framework vs. .NET 6+

| Topic | .NET Framework 4.6.2+ | .NET 6/7/8+ |
|-------|----------------------|-------------|
| PDF renderer package | Included in `DevExpress.Document.Processor` | Also requires `DevExpress.Pdf.SkiaRenderer` |
| `Process.Start()` | `Process.Start("file.docx")` works directly | Must set `UseShellExecute = true` |
| Assembly references | Can reference DLLs directly from installer | NuGet packages strongly recommended |
| `System.Drawing` | Available by default | Requires `<UseWindowsForms>true</UseWindowsForms>` in `.csproj` or use `DevExpress.Drawing` equivalents |
| Linux/macOS printing | Not supported | Supported via `DXPrinterSettings` and CUPS |
| Target framework in `.csproj` | `<TargetFramework>net462</TargetFramework>` | `<TargetFramework>net8.0</TargetFramework>` |

## Using `System.Drawing.Color` on .NET Framework

On .NET Framework, `System.Drawing` is available without extra configuration:
```csharp
using System.Drawing;

cp.ForeColor = Color.DarkBlue;
cp.BackColor = Color.LightYellow;
```

## Export to PDF (.NET Framework)

PDF export works the same way on .NET Framework — use `ExportToPdf`:

```csharp
using DevExpress.XtraRichEdit;
using DevExpress.XtraPrinting;

using (var server = new RichEditDocumentServer())
{
    server.LoadDocument("input.docx", DocumentFormat.Docx);

    PdfExportOptions options = new PdfExportOptions();
    options.DocumentOptions.Author = "My Application";
    options.Compressed = true;

    server.ExportToPdf("output.pdf", options);
}
```

## Printing on .NET Framework

```csharp
using System.Drawing.Printing;

using (var server = new RichEditDocumentServer())
{
    server.LoadDocument("input.docx", DocumentFormat.Docx);

    PrinterSettings settings = new PrinterSettings();
    settings.PrinterName = "Microsoft Print to PDF";
    settings.Copies = 2;

    server.Print(settings);
}
```

## ASP.NET (Classic) Usage

For ASP.NET WebForms or MVC (classic), use `RichEditDocumentServer` in a controller or code-behind — it is fully server-safe and has no UI dependencies:

```csharp
// In an MVC action or WebForms code-behind
using (var server = new RichEditDocumentServer())
{
    server.LoadDocument(Server.MapPath("~/App_Data/template.docx"), DocumentFormat.Docx);
    // Modify ...

    // Return as download
    Response.ContentType = "application/vnd.openxmlformats-officedocument.wordprocessingml.document";
    Response.AddHeader("Content-Disposition", "attachment; filename=output.docx");
    server.SaveDocument(Response.OutputStream, DocumentFormat.Docx);
}
```

## Troubleshooting (.NET Framework Specific)

- **`TypeLoadException`**: Ensure all DevExpress DLLs are the same version and that none are missing.
- **`FileNotFoundException` for DevExpress DLLs**: Check that the assembly is in the output directory or the GAC.
- **Mixed version errors**: Use NuGet to manage versions rather than manually referencing DLLs.
- **`System.Drawing` not found**: Add a reference to `System.Drawing.dll` in the project references.
